CSS

CSS (Cascading Style Sheet) adalah mekanisme sederhana yg mengatur gaya/style pada halaman web. ANATOMY CSS/struktur : - selector { property: value; }. Contoh h1 { color: blue; } . Artinya css pilih seluruh elemen h1 lalu ubah text didalamnya menjadi warna biru
  1. embed = disimpan di html yg sama dibagian head menggunakan tag style
  2. inline = didalam file yg sama dan di baris html itu juga, exp: p style = "color: blue;"
  3. external = disimpan pada bagian head untuk meload css yg diluar, exp: tag link href="" rel=""
  1. font-family = utk mengatur jenis font yg akan digunakan. exp: font-family: Arial, Helvetica, sans-serif;
  2. font-size = utk mengatur ukuran besar kecilnya font. exp: font-size: 30px;
  3. font-weight = utk mengatur ketebalan font. exp: font-weight: 800;
  4. font-variant = mengubah font menjadi small caps. exp: font-variant: small-caps;
  5. font-style = mngubah font menjadi bercetak miring. exp: font-style: italic;
  6. line-height = mengatur spasi antar baris. exp: line-height: 2px;
  1. color= memberi warna. exp: color: red;
  2. text-align= utk mengatur rata tulisan. exp: text-align: center;
  3. text-indent= memberi identasi text / paragraf pertama atau tab. exp: text-indent: 20px;
  4. text-decoration = mengatur dekorasi text. exp: text-decoration: underline;
  5. text-transform= mengubah huruf menjadi huruf besar,kecil/kapital. exp: text-transform: uppercase;
  6. letter-spacing= mengatur spasi antar huruf. exp: letter-spacing: 10px;

CSS Layouting

Css Layouting adalah teknik css untuk mengatur dan meletakkan atau memposisikan elemen-elemen html yang sudah dibuat menggunakan sintaks css.

Display = mengatur perilaku dari kotak di CSS value pada display : inline, inline-block, block, none

Block
- elemen HTML yang secara default menambahkan baris baru ketika dibuat
- jika tidak diatur lebar-nya, maka lebar default dari elemen block akan memenuhi lebar dari browser/parent-nya
- kita dapat mengatur tinggi dan lebar dari elemen block
- didalam elemen block, kita dapat menyimpan tag dengan elemen inline,inline-blockatau bahkan elemen block lagi
contoh elemen block: h1-h6, p, ol, ul, li, form, hr, div

inline(tidak menambah baris baru/disebelah kanan):
- elemen html yg secara default tidak menambahkan baris baru ketika dibuat
- lebar dan tinggi elemennya sebesar konten yang ada di dalamanya
- kita tidak dapat mengatur tinggi dan lebar elemen inline
- margin dan padding hanya mempengaruhi elemen secara horizontal, tidak vertical
elemen inline : b, stong, u, em, a, span, sub, sup, button, input, label, select, textarea

Inline-block
- tidak ada elemen yang secara default memilki properti display: inline-block;
- kita harus ubah secara manual properti tersebut
- perilaku dasar sama dengan elemen inline
- perbedaannya, elemen inline-block dapat kita atur tingi dan lebar-nya

exp: ubah daftar link dari li displaynya block menjadi inline
.navigasi li { display: inline; }
exp: utk menghilangkan gambar
img { display: none; }

Dimensi itu berisi tentang width dan height
Dimensi:
px = satuan absoulute/ nilainya pasti
% = nilai relative, relative terhadap elemen parentnya atau pembungkusnya
overflow = utk mengatur perilaku elemen yg keuar dari parentnya value overflow : visible, auto, hidden, scroll

CSS box model mendefinisikan 'kotak' yang dihasilkan oleh sebuah elemen, lalu menampilkannya sesuai dengan format visualnya
CSS box model terdiri dari margin, border, padding dan content
- margin = area transparan disekitar kotak(diluar border)
- border = batas disekeliling padding
- padding = area transparan didalam kotak(antara conten dan border)
- content = konten sebenarnya didalam box, bisa berupa text atau gambar
properti: - margin : margin-top, right, bottom, left, margin, begitupun yang lain

auto = nilai yg bisa kita kasih khusus utk margin kanan dan kiri maka kotak tersebuat akan ditengah.
exp: margin-left: auto; margin-right: auto;

-px = negatif margin. itu mengikuti arah/menerobos
exp: margin-top: -50px; margin-left: -100px;

overlapping margin = terjadi ketika kita menggabungkan kedua buah margin kiri dg kanan atau atas dg bawah
- margin bagian vertikal dia tidak menambahkan yg diambil margin yg paling besar
- margin horizontal = marginnya ditambahkan keduanya

padding = memeberikan jarak antara sebuah kotak dengan tulisan perbedaan padding dg margin :
- padding tidak bisa diberi niali negatif
- padding gak punya nilai auto
- padding menambah ukuran elemen
border juga menambah ukuran elemen, yaitu => total dimensi = width/height + padding + border

Box sizing adalah properti yang menerima nilai padding dan border pada suatu elemen termasuk sebagai nilai total dari width dan height suatu elemen
exp: div { width: 600px; margin:10px auto; box-sizing: border-box; }
.satu { /* 600 - [2*10] - [2*5] = 570px width: 570px;*/ background-color: royalblue; padding: 10px; border: 5px solid red; }

Float = properti pada CSS untuk mengatur posisi sebuah elemen, sebuah elemen dapat dipaksa untuk berada disebelah kiri atau kanan dari parent/pembungkusnya dengan menggunakan properti ini

- out-of flow

CSS Static
- static = nilai default dari tiap-tiap elemen ketika tidak diberi properti position
- menggunakan position selain static (non-static), akan membuat sebuah elemen menjadi seolah-olah berbeda dimensi dari elemen lainnya
- elemen yg diberi position selain static dapat menggunkan properti top,left,bottom dan right untuk mengatur posisinya

CSS Relative
yg dikasi nilai position relative maka div itu akan maju 1 dimensi
- ketika kita menggerakkan elemen dg posisi relative (menggunakan properti top,left,bottom,right)ruang yg ditempati oleh elemen tsb masi ada
- ketika kita menggerakkan elemen dg posisi relative, elemen akan bergerak relatif thd posisi semulanya
- jika kkita memberi properti top: 0; dan left:0; maka elemen tdk berubah posisinya

Absolute
- ketika kita menggerakkan elemen dg posisi absolute(menggunakan properti top,left,bottom,right),ruang yg ditempati oleh elemen tersebut dianggap tidak ada
- ketika kita menggerrakkan elemen dg posisi absolute, elemen akan bergerak relatife thd posisi dari elemen parent-nya, selama elemen parent memiliki elemen non static
- jika kita memberi properti top: 0; dan left: 0; maka elemen akan berada diujung kiri atas elemen parent-nya

Fixed
selalu bergerak relative ke window browser atau bersifat tetap terhadap browser tanpa adanya perubahan bahkan pada saat di scroll

z-index adalah property css yang mulai ada pada css2, fungsinya yaitu untuk mengatur posisi depan atau belakang suatu elemen. Value untuk property z-index berupa angka. Minimum value adalah 0. Tapi untuk browser-browser dengan versi lama bisa berupa angka negatif.
z-index = utk mengatur tumpukan dari elemen ketika elemen non-static, max=9999 & min= -9999. exp: z-index: 1

Flexbox

Flek box = mempermudah utk mengatur tata letak css/div definisi Flek box adalah sebuah model layout 1 dimensi yg dapat mengatur jarak dan penjajaran antar item dalam sebuah container.
  • 1 dimensi ? maksudnya dg menggunakan flek box kita hanya dapat mengatur 1 dimensi pada saat tertentu, antara baris(hor) atau kolom(ver), tidak bisa keduanya sekaligus.
  • 2 dimensi menggunkan css grid flexbox layout module adalah sebuahh modul yg menawarkan cara yg efektif utk menyusun, mensejajarkan dan mendistribusikan jarak antar item di dalam sebuah container, meskipun ukurannya dinamis atau bahkan kita tidak tau.

main axis = set elemen secara horizontal
cross axis = set elemen secara vertikal
main size = ukuran dari containernya
cross size = ukuran dari kolom
main start - main end = menentukan awal dan akir dr elemen secara hor
cross start - cross end = menentukan awal dan akir dr elemen secara ver
istilah pada flexbox :
- main axis = sumbu utama dari sebuah container yg menentukan urutan dari penempatan items secara horizontal
- main-start | main-end = mulai dan berakhirnya items yg disimpan di dalam container
- main size = ukuran (widt/height) dari container yg akan membuat dimensi dari itemsnya relatif terhadap main size

.container { display: flex; /* atau inline-flex */ }
membuat sebuah elemen parent menjadi flex box, dan membuat elemen2 di dalamnya bisa berprilaku flex juga
flex-direction untuk mengatur arah dari urutan dari elemen2 didalamnya
.container { flex-direction: row | row-reverse | column | column-reverse; }
row = dari kiri kekanan, reverse = kebalikannya
column = atas kebawah
flex-wrap = utk mengatur ketika elemen yg kita punya itu sudah tdk muat lagi ke container dalam barisnya, lalu kasih turun dibawahnya
.container { flex-wrap: nowrap | wrap | wrap-reverse; }
secara default semua item didalam container akan berada pada satu baris meskipun ukurannya sudah tidak cukup, wrap memungkinkan untuk memindahkan item ke baris dibawahnya

.container { jusitify-content: filex-start | fixed-end | center | space-between | space-around | space-evenly; }
mengatur jarak antar items secara horizontal
.container { align-items: flex-start | flex-end | center | stretch | baseline; }
mengatur perilaku penjajaran items dengan arah vertikal
.container { align-content: flex-start | flex-end | center | space-between | space-around; }
mengatur jarak antar items terhadap sumbu horizontal (hanya berfungsi ketika items lebih dari 1 baris)

Property pada items
order = untuk mengatur posisi pada items
contoh => .satu { order: 2}
flex-grow / flex untuk mengatur ukuran dari itemnya
contoh => .satu { flex-grow: 1; } .dua { flex-grow: 3; }
align-self untuk mengatur perilaku penjajaran sebuah item yg spesifik terhadap kusus untuk 1 kolom sj
.item { align-self: auto | flex-start | flex-end | center | baseline | stret; }

Grid

Secara sederhana CSS Grid adalah CSS yang dapat membagi kolom pada website menjadi beberapa bagian sesuai yang diinginkan, baik secara horizontal maupun vertikal. Penggunaan CSS Grid bertujuan untuk mempermudah developer untuk membuat layout dari design yang telah dibuat

exp:
.container { width: 600px; margin: 50px auto; border: 5px solid #333; box-sizing: border-box; padding: 5px; display: grid; grid-template-areas: "header header" "aside main" "nav main" "footer footer"; grid-template-columns: 1fr 1.5fr; /* fr/frektion = pecahan */ grid-template-rows: 1fr 1fr 1.3fr; }
header { grid-area: header; background-color: #B063D8; display: flex; justify-content: center; align-items: center; }

Grid container = elemen pembungkus grid, penulisanan => display:grid;
Grid item = element2 yg berada 1level didalam grid container
Grid line = garis horizontal(kolom) atau vertikal(baris) yg memisahhkan grid menjadi beberapa bagian dan ditandai dg angka
Grid cell = perpotongan/pertemuan antara baris dan kolom didalam grid
Grid area = kumpulan lebih dari satu grid cell yg membentuk kotak
Grid track = ukuran/jarak antara 2 grid line, bisa horizontal(kolom) atau vertikal(baris)
Grid gap = jarak antara grid track/cell

mendefinisikan kolom/baris dg cara menuliskan nilai dipisahkan oleh spasi. nilai mempresentasikan ukuran grid track dan spasi mempresentasikan grid line
sintaks:
.container{
display: grid;
grid-template-columns: track-size ...| line-name track-size ...;
grid-template-rows: track-size ...| line-name track-size ...;
- track-size: bisa berupa px, %,auto, atau fr(ftaction/bagian)
- line-name: nama yg bisa diberikan pada track
}
exp:
.container { width: 600px; margin: 50px auto; /* utk ketengah */ display: grid; grid-template-columns: 25% auto 50px; grid-template-rows: 50px 100px auto; }

Explicit = secara jelas menjelaskan ukuran grid
Implicit = ukuran otomatis ditentukan oleh grid
grid-auto-columns & grid-auto-rows = mengatur ukuran grid track yg tidak dituliskan pada grid-template(implicit track)
.container { grid-auto-columns: track-size; grid-auto-rows: track-size; }
gri-auto-flow = mengatur penempatan item/cell pada grid track, termasuk yg ditulis secara implicit
.container{ grid-auto-flow: row| column| row dense| column dense; }
exp:
.container { width: 600px; margin: 50px auto; display: grid; grid-template-columns: 50px 200px; grid-template-rows: 100px 50px; /* implisit track */ grid-auto-columns: 400px; grid-auto-rows: 400px; }

exp:
.container {
width: 600px;
margin: 50px auto;
display: grid;
grid-template-columns: 50px 200px;
grid-template-rows: 100px 50px;
/* Auto Flow */
grid-auto-flow: column;
}

repeat() = menentukan ukuran grid track secara langsung
exp: .container { width: 60%; margin: 50px auto; display: grid; grid-template-columns: repeat(9, 1fr) repeat(2, 2fr); }

min-content & max-content = menetukan seberapa besar ukuran grid track berdasarkan content pada sebuah item
min = memecah ukuran kalimat per kata
max= membuat kalimat tsb pjg
exp: .container { width: 60%; margin: 50px auto; display: grid; grid-template-columns: 1fr min-content 1fr; /* min/max-content */ }

auto-fill & auto-fit adalah menentukan jumlah item utk berada pada grid track
exp: .container{ display: grid; grid-template-columns: repeat(auto-fit, 150px)

minmax() = utk menentukan ukuran min dan maxdari grid track

Grid Template Area = mendefinisikan grid template menggunakan nama dari area yg ditulis pada property gri-area pada item
.container { grid-template-areas: "nama-area> | none | '.'" "nama-area> nama-area>" }
exp: grid-template-areas: 'header header header' 'sidebar sidebar sidebar' 'content content content' 'footer footer footer';

Grid Template = shorthand utk mendefinisikan grid-template-rows, grid-template-columns dan grid-template-areas dalam satu deklarasi
.container { grid-template: grid-template-rows> | grid-template-area> / grid-template-columns>; }
exp: grid-template: [baris1-mulai] 'header header header' 50px [baris1-akhir] [baris2-mulai] 'footer footer footer' 50px [baris2-akhir] / 1fr 2fr 3fr;

grid-gap / gap = shorthand utk mendefinisikan column-gap, row-gap dalam satu deklarasi
grid = shorthand utk mendefinisikan properti2 berikur dalam satu deklarasi (grid-template-rows, grid-template-columns, grid-template-areas, grid-auto-rows, grid-auto-rows, dan grid-auto-flow)
exp: grid-column-gap: 10px; grid-row-gap: 25px;

Justify content adalah mengatur posisi seluruh grid container pd sumbu horizontal.
ini bisa dilakukan ketika ukuran total grid lebih kecil dari ukuran containernya, biasanya ketika grid itemsnya menggunakan ukuranyg fixed(px)
exp: .container { justify-Content: start|end|center|stretch|space-around|space-between|space-evenly; }

Place Content = shorthand utk menulis justify-content dan align-content dalam satu deklarasi
.container { place-content: align-content> justify-content>; }
exp: /* justify-content: center; align-content: center; */ place-content: space-between;

Place items = shorthand utk menulis justify-items dan align-items dalam satu deklarasi
.container { place-items: align-items> justify-items>; }
exp: /* justify-items: center; align-items: center; */ place-items: center end;

grid-column-start & grid-column-end | grid-row-start & grid-row-end = menentukan tempat dari grid item dg mengacu pd nomor atau nama grid line yg spesifik
.item { grid-column-start: nomor>|nama>|span nomor>| span nama>|auto; .. yg lain juga sama
ketereangan: nomor>|nama>: mengacu pd nomor atau nama yg kita beri pd grid line
span nomor>: item akan memanjang/melebar sampai sejumlah nomor
span nama>: item akan memanjang/melebar sampai ke nama grid line tertentu
exp: grid-column-start: -1; grid-column-end: -2; grid-row-start: -4; grid-row-end: -1;

grid-column & grid-row = shorthand utk start dan end pada grid-column & grid-row
.item { grid-column: start-line>/end-line>|start-line>/spanvalue>; }
exp: grid-column: 1 / span 2; grid-row: 1 / span 2;

grid-area = menentukan nama area pd item sesuai dg template yg sudah dibuat sebelumnya memlalui properti grid-template-areas bisa digunakan sbg shorthand utk grid-column-start, grid-column-end, grid-row-start dan grid-row-end
.item { grid-area: name>|row-start>/column-start>/row-end>/column-end> } exp:
grid-template-areas: 'header header header' 'sidebar content content' 'footer footer footer';

.item { place-self: align-self> justify-self>; }
exp: .my-item { background-color: lightblue; place-self: end; }

Sass

SASS = Sintaktikali Awesome Style Sheet. Alasan menggunakan Sass: mempermudah penulisan, pengelolaan, memiliki banyak fungsi built-in, dapat membuat fungsi sendiri, dapat memodifikasi framework css(cth: boostrap), banyak perusahaan yg menggunakan SASS

Sass vs Scss = kalo Scss sintaksnya hampir mirip dg css (keduanya akan tetap di compile).
instalasi compiler SASS: aplikasi, commad line, vscode extension
Alasan tidak menggunakan Sass: - banyak fitur sass yg sekarang sudah dimiliki oleh css, - tidak utk projec kecil, - styled component/css in js
exp: $myColor: salmon; $pColor: #fff; header { background-color: $myColor; h1 { font-size: 3rem; } p { color: $pColor; } }

Nesting adalah membuat hirarki visual yg jelas utk selector css kita
& = parent selector
container ini ada diatas dari class main/ elemen parent
.main { .container & { background-color: green; }
// jika pake elemen lebih baik manggil pake class
.footer { & &-heading { font-size: 2em; background-color: yellow; }

Kelebihan variable di CSS memudahkan, kekurangan belum semua browser support
:root {
:root agar elemennya dapat mengakses variabel ini
--primary-color: #ababab;
--secondary-color: pink;
--text-color: red;
}
$primary-color: #ababab; // variabel local
$primary-color: #ababab !global; //membuat variable menjadi scope global
Map => exp: $alert-danger: red;
atau $alert: ( 'danger': red,
pemanggilan : color: map-get($alert, 'danger');

mixin adalah kumpulan style yg dapat kita gunakan berulang-ulang keyword wajib @mixin = utk membuat mixin, @include = memanggilnya
membuat mixin, nama mixin bebas
exp buat: @mixin inline-list { margin: 0; padding: 0; list-style: none; li { display: inline-block; } }
exp panggil : header ul { // panggil mixin @include inline-list(); li { background-color: black; padding: 0.7em; a { color: white; } } }

modules = membuat code sass kita modular/mudah utk dikelola keywordnya @import & @use
agar file main dapat mengenali file _mixin, maka panggil import sudah siadopsi oleh css biasa, nama lengkap dan extensinya
kekurangan @import : variablenya menjadi global, dll dapat dilihat di website sass
exp : @use 'modules/header'; dan @mixin inline-list { }

flow control = sebuah konsep utk mengatur alur kendali dari script SASS
keywordnya :
1. @if, @else. @else if = pengkondisian/percabangan
2. @for = utk perulangan
3. @while = perulangan
4. @each = perulangan
exp: @mixin color-mode($light-mode: true) { @if $light-mode { background-color: $light-bg; color: $light-text; } @else { background-color: $dark-bg; color: $dark-text; } }
